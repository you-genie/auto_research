# Claude Code 신기능 빠른 참조 가이드

**핵심만 빠르게 학습하는 1-2페이지 요약**

---

## 🤖 Agent Teams (멀티 에이전트 협업)

### 한 문장 설명
"여러 AI 에이전트가 각자 독립적인 컨텍스트로 작업하면서 공유 작업 목록으로 협력"

### 5분 안에 이해하기

```
리더 세션 (당신)
   ↓ (작업 분배)
┌─────────┬─────────┬─────────┐
│Teammate1│Teammate2│Teammate3│
│   (보안) │  (성능) │  (테스트)│
└────┬────┴────┬────┴────┬────┘
     ↓         ↓         ↓
  공유 작업 목록 & 메시지 채널
```

### 언제 쓸까?

✅ **쓸 때:**
- 병렬 코드 리뷰 (보안/성능/테스트 동시 검토)
- 경쟁 가설 조사 (여러 이론 동시 검증)
- 크로스레이어 리팩토링 (DB/API/프론트엔드 병렬)

❌ **쓰지 말 때:**
- 순차 의존성이 강한 작업
- 같은 파일을 여러 에이전트가 수정하는 경우
- 시간이 충분한 경우 (비용이 3-5배)

### 빠른 시작

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

# 세션에서:
"병렬 코드 리뷰 팀을 만들어줄래.
보안, 성능, 테스트 커버리지 리뷰어 각 1명씩.
PR #142 검토해줄래."

# 팀원 간 이동
Shift+Down  → 다음 팀원으로 이동
Shift+Up    → 이전 팀원으로 이동

# 작업 완료 후 정리
"팀 정리해줄래"
```

### 비용 계산

```
3명 팀, 100시간 작업:
- 단일 세션: 80만 토큰 = $0.12
- Agent Team: 140만 토큰 = $0.21
- 추가 비용: $0.09 (~2시간 절감)
→ 시간 가치 > $45/시간이면 추천
```

### 주의사항

⚠️ **주의:**
- 팀원끼리 같은 파일 수정 금지 (충돌 위험)
- `/resume` 후 팀원 손실 (새로 생성해야 함)
- 세션 종료 시 모든 팀 삭제
- 한 세션에 한 팀만 (정리 후 새 팀 생성)

## ⏰ Scheduled Tasks (반복 자동화)

### 한 문장 설명
"정해진 시간에 프롬프트를 자동으로 반복 실행"

### 5분 안에 이해하기

```
3가지 스케줄링 방식:

┌─────────────────────────────────┐
│ Session Task (/loop)            │
│ - 현재 세션에서만 실행          │
│ - 세션 종료 시 삭제             │
│ - 임시 폴링용 (배포, PR 감시)   │
│ - 3일 자동 만료                 │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ Desktop Task                    │
│ - 로컬 머신에서 실행            │
│ - 세션 종료 후에도 실행         │
│ - 로컬 파일 접근 가능           │
│ - 머신이 켜져있어야 함          │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ Cloud Task                      │
│ - Anthropic 클라우드에서 실행    │
│ - 24/7 자동화 가능              │
│ - 머신 켜짐 불필요             │
│ - 로컬 파일 접근 불가           │
└─────────────────────────────────┘
```

### Cron 표현식 30초 학습

```
분 시간 일 월 요일
*  *   *  *  *

# 자주 쓰는 패턴:
*/5 * * * *       → 5분마다
0 * * * *         → 매시간 (정각)
0 9 * * *         → 매일 9:00 AM
0 9 * * 1-5       → 평일 9:00 AM
0 0 * * 0         → 매주 일요일 자정

# /loop로는 간단하게:
/loop 5m 배포 확인
/loop check every 2 hours
remind me at 3pm to push
in 30 minutes, test completed?
```

### 언제 뭘 쓸까?

| 상황 | 추천 |
|------|------|
| 배포 상태 5분마다 확인 | Session (`/loop 5m`) |
| 매일 밤 테스트 실행 | Desktop (로컬 접근 필요) |
| 매일 정각 보고서 생성 | Cloud (24/7) |
| PR 리뷰 완료 대기 | Session (`/loop 15m`) |
| 2시간 뒤 상기 | Session (`in 2h`) |

### 빠른 시작

```bash
# 세션 작업 (가장 간단)
claude
/loop 5m 배포 상태 확인해줄래

# 작업 확인
사용자: "스케줄된 작업이 뭐가 있어?"

# 작업 취소
사용자: "배포 확인 작업 취소해줄래"
```

### 비용 계산

```
배포 감시 (5분마다, 1시간 = 12회):
- 회당 2K 토큰
- 시간당: 24K 토큰 = $0.004
- 일당: 576K 토큰 = $0.09
- 월당: 17.3M 토큰 = $2.60

최적화: 간격 10분 → 5분 비용으로 절감
```

### 주의사항

⚠️ **주의:**
- Session 작업: 3일 후 자동 만료
- Catch-up 없음: 기한 지난 실행은 스킵
- 에러 재시도 없음: 실패해도 계속 실행 (수동 관리)
- 에러 추적 필수: 실패한 작업 모니터링

## 🆚 어떤 걸 쓸까?

### 의사결정 트리

```
큰 병렬 작업인가? (여러 팀원 필요)
├─ YES → Agent Teams
│   - 코드 리뷰
│   - 리팩토링
│   - 복합 분석
│
└─ NO → 반복 자동화인가? (정기적으로 실행)
    ├─ YES → Scheduled Task
    │   - 배포 감시
    │   - PR 모니터링
    │   - 정기 리포트
    │
    └─ NO → 빠른 도움인가?
        ├─ YES → Subagents 사용
        ├─ NO → 단일 세션 사용
```

### 비교표

| 특성 | Agent Teams | Tasks | Subagents |
|------|------------|-------|-----------|
| **협력** | ✅ 높음 | ❌ | ❌ |
| **비용** | $$$ | $ | $$ |
| **시간** | 빠름 | 정기적 | 중간 |
| **복잡도** | 높음 | 낮음 | 중간 |
| **학습곡선** | 가파름 | 낮음 | 중간 |
