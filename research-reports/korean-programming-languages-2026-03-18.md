## 개요

한글 프로그래밍 언어는 **프로그래밍의 모든 요소(키워드, 변수명, 문법)를 한글로 작성할 수 있는 프로그래밍 언어**를 의미한다. 

### 왜 한글 프로그래밍 언어가 필요한가?

- **진입 장벽 감소**: 영어 모르는 초보자도 프로그래밍 학습 가능
- **모국어 사고**: 자연스러운 한국어 문법으로 알고리즘 표현
- **문화 자산**: 한글의 과학적 설계를 프로그래밍에 활용
- **포용성**: 다양한 언어권에서의 프로그래밍 접근성 확대

### 핵심 과제

- **가독성 vs 타이핑 효율성**: 한글은 영문자보다 타이핑 횟수가 많음
- **기존 생태계와의 호환성**: 대부분의 라이브러리와 도구가 영문 기준
- **커뮤니티 규모**: 사용자 규모가 작아 학습 자료 부족
- **유지보수**: 개발자 이탈로 인한 장기 지원 어려움

### 1. **Han (한)** ⭐ 최신 (2026년 출시)

#### 개요
- **개발자**: xodn348 (개인 프로젝트)
- **출시 시기**: 2026년 3월 (최근 출시)
- **상태**: 🔴 **활발히 진행 중 (최신)**
- **특징**: Rust로 작성된 **정적 타입 컴파일 언어**

#### 기술 스펙

| 항목 | 설명 |
|------|------|
| **컴파일 방식** | LLVM IR을 통한 네이티브 바이너리 생성 |
| **실행 방식** | 컴파일 모드 + 인터프리터 모드 + REPL |
| **타입 시스템** | 정적 타입 체킹 (5가지 기본 타입) |
| **기본 타입** | 정수(정수), 실수(실수), 문자열(문자열), 불(불), 없음(없음) |

#### 주요 특징

**1) 한글 키워드 (모두 실제 한국어)**
```
함수: function 정의
만약: conditional
반환: return
변수: variable
반복: for loop
동안: while loop
```

**2) 첫 번째 한글 컴파일 언어**
- 모든 키워드와 식별자가 한글
- LLVM 통해 native 바이너리로 컴파일
- 트리워킹 인터프리터도 제공

**3) 핵심 기능**
- ✅ 배열 (음수 인덱싱 지원)
- ✅ 구조체(구조)와 impl 블록
- ✅ 클로저/람다 함수
- ✅ 패턴 매칭 (맞춤)
- ✅ 예외 처리 (시도/처리)
- ✅ HashMap/사전
- ✅ 정규식
- ✅ JSON 파싱
- ✅ HTTP 통신
- ✅ 파일 I/O

#### 문법 예제

**Hello World**
```han
출력("안녕하세요, 세계!")
```

**변수 선언 및 함수**
```han
변수 나이: 정수 = 25
변수 이름: 문자열 = "홍길동"

함수 더하기(가: 정수, 나: 정수) -> 정수 {
  반환 가 + 나
}
```

**제어문**
```han
만약 점수 >= 90 {
  출력("A")
} 아니면 만약 점수 >= 80 {
  출력("B")
} 아니면 {
  출력("C")
}
```

**반복문**
```han
반복 변수 i = 0; i < 10; i += 1 {
  출력(i)
}
```

**구조체**
```han
구조 사람 {
  이름: 문자열,
  나이: 정수
}

변수 p = 사람 { 이름: "홍길동", 나이: 30 }
```

### 2. **약속 (Yaksok)** ⭐ 가장 실용적

#### 개요
- **개발자**: 하재승, Puzzlet Chung
- **출시 시기**: 2015년
- **상태**: 🟢 **계속 개발 중**
- **특징**: **Python 3 기반의 인터프리터 언어**
- **공식 홈페이지**: http://yaksok.org/

#### 개발 배경

하재승 개발자는 다음과 같이 설명했다:
> "프로그래밍을 처음 접하는 사람에게 친근감을 주고 싶었다. '약속'이라는 단어는 다른 언어와 겹치지 않고, 발음이 좋으며, 의미도 좋아서 선택했다."

#### 핵심 특징

**1) 자연스러운 한국어 문법**
- Python 문법에 한국어를 정밀하게 매핑
- 상용 소프트웨어, 웹 애플리케이션, 게임, 앱까지 개발 가능
- 한글 변수명, 함수명, 클래스명 모두 지원

**2) 문법 설계 원칙**
- 한국어의 자연스러운 문법 유지
- 동사 + 목적어 구조 활용
- 조사를 이용한 명확한 인자 전달

#### 문법 예제

**기본 변수 선언**
```yaksok
정수 가, 나, 다
실수 키
문자 이름
```

**함수 정의**
```yaksok
(키 "키" 몸무게 "몸무게"의 비만도 결과) {
  결과 = 몸무게 / (키 * 키)
}

키 1.77 몸무게 68의 비만도 비만도
```

**제어문**
```yaksok
만약 조건 {
  ...
} 아니면 {
  ...
}
```

- ✅ 교육용 프로그래밍 학습
- ✅ 간단한 게임 개발
- ✅ 자동화 스크립트
- ✅ 웹 크롤링

### 3. **누리 (Nuri)** ⭐ 함수형 프로그래밍

#### 개요
- **개발자**: 서동휘 (suhdonghwi)
- **출시 시기**: 2020년
- **상태**: 🟡 **유지보수 중 (정식 릴리즈 전)**
- **특징**: **함수형/선언형 한글 프로그래밍 언어**
- **공식 저장소**: https://github.com/suhdonghwi/nuri

#### 핵심 철학

누리는 명령형(imperative)이 아닌 **선언형(declarative)** 패러다임을 따른다.

> "선언형 프로그래밍이 자연어에 더 적합하다. 평소에 개념을 설명할 때 '이것은 이거고, 저것은 저거다'라는 방식으로 정의를 해나가며 설명한다. 이것이 선언형 프로그래밍이다."

#### 독특한 특징

**1) 한국어 문법의 활용: 조사**

한국어의 조사를 프로그래밍에 활용하여:
- 함수 인자의 위치를 조사로 명확히 함
- 함수 호출의 순서를 바꿔도 의미 변하지 않음

```nuri
동사 [ㄱ]을 [ㄴ]으로 나누다: [ㄱ] / [ㄴ]

10을 5로 나누다 == 5로 10을 나누다 == 2
```

**2) 함수형 프로그래밍의 3가지 타입**

```nuri
# 일반 함수
함수 [목록]의 합계:
  만약 [목록] == {} 이라면 0
  아니라면 [목록]의 첫번째 + ([목록]의 나머지)의 합계

# 동사: 연쇄 호출 가능
동사 [ㄱ]을 [ㄴ]으로 나누다: [ㄱ] / [ㄴ]
동사 [ㄱ]과 [ㄴ]을 더하다: [ㄱ] + [ㄴ]

10을 5로 나누고 출력하다        # 출력: 2
2와 8을 더하고 5로 나누고 출력하다 # 출력: 2

# 형용사: 특징 판정
형용사 [ㄱ]과 [ㄴ]이 같다: [ㄱ] == [ㄴ]
```

**3) 부분 호출(Currying) 지원**

```nuri
동사 [ㄱ]과 [ㄴ]을 더하다: [ㄱ] + [ㄴ]

5와 10을 더하다 == 15
10을 더하다 == (함수) # 하나의 인자를 받아 거기에 10을 더하는 함수

{1, 2, 3}에 (2를 더하다)를 각각 적용하다 == {3, 4, 5}
```

**4) 변수 없음 (상수만 존재)**

선언형 언어의 특징으로 부수 효과(side effects)를 피함.
- 재귀 함수를 통해 모든 프로그래밍 구현 가능

#### 문법 예제

```nuri
# "안녕"과 "세상!"을 연결하고 보여주다
"안녕" 과 "세상!"을 연결하고 보여주다
```

#### 장점

✅ 한국어 문법의 깊은 활용

### 4. **아희 (Aheui)** ⭐ 가장 유명한 난해 언어

#### 개요
- **개발자**: Puzzlet Chung (유의영)
- **출시 시기**: 2008년
- **상태**: 🟢 **커뮤니티 유지 중**
- **특징**: **한글의 시각적 특징을 활용한 난해 프로그래밍 언어**
- **공식 문서**: https://aheui.readthedocs.io/

#### 개발 배경

Aheui는 **에소테릭 프로그래밍 언어(Esoteric Language)**다.
- 실용성보다는 예술적 가치와 한글의 아름다움에 초점
- "한글 낱자의 디자인을 살릴 수 있는 프로그래밍 언어를 만들고 싶었다"

#### 핵심 특징

**1) 한글 구조를 그대로 활용**

한글의 자음과 모음이 정확히 명령어와 방향을 지정:
- 초성(자음): 실행 명령어 (덧셈, 뺄셈, 스택 조작 등)
- 중성(모음): 커서의 이동 방향 (↑, ↓, ←, →)
- 종성(자음): 명령어의 매개변수

```
'ㅏ', 'ㅓ', 'ㅗ', 'ㅜ' → 커서 방향 지정
'ㄷ' → 덧셈 (addition)
'ㄸ' → 곱셈 (multiplication)
'ㄴ' → 나눗셈 (division)
```

**2) 2차원 그리드 기반 실행**

Befunge와 유사하게 2D 그리드에서 커서(instruction pointer)가 이동:
```
밤밣따빠밣밟따뿌
빠맣파빨받밤뚜뭏
돋밬탕빠맣붏두붇
```

**3) 28개의 스택/큐 구조**

프로그램 상태 관리:
- 26개의 스택 (각 자음 + 빈 자음)
- 1개의 큐
- 1개의 확장 프로토콜

#### 문법 예제

**기본 연산**
```aheui
# "Hello, world!" 출력 (전체 코드)
밤밣따빠밣밟따뿌
빠맣파빨받밤뚜뭏
돋밬탕빠맣붏두붇
볻뫃박발뚷투뭏붖
뫃도뫃희멓뭏뭏붘
뫃봌토범더벌뿌뚜
뽑뽀멓멓더벓뻐뚠
뽀덩벐멓뻐덕더벅
```

**주요 명령어**
```
ㅇ: null operation (아무것도 하지 않음)
ㅎ: terminate (프로그램 종료)
ㄷ: addition (덧셈)
ㄸ: multiplication (곱셈)
ㄴ: division (나눗셈)
ㅌ: subtraction (뺄셈)
ㄹ: modulo (나머지)
ㅁ: pop/print (스택에서 꺼내기)
ㅂ: push (스택에 넣기)
ㅅ: storage selection (저장소 선택)
ㅈ: compare (비교)
ㅊ: fork (조건부 분기)
```

## 기술 심화 분석

### Han의 기술 스택

```
Source (.hgl)
    ↓
┌─────────────────────────────────┐
│ Lexer (어휘 분석)                │
│ 한글 토큰 인식                    │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ Parser (구문 분석)               │
│ 재귀적 하강 파서                  │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│ AST (추상 구문 트리)             │
└──────────────┬──────────────────┘
           ┌───┴────┐
           ↓        ↓
      ┌─────────┐  ┌──────────┐
      │Interpreter│ │ CodeGen  │
      │직접 실행   │ │LLVM IR   │
      └────┬────┘  └────┬─────┘
           ↓            ↓
         결과      clang → Binary
```

### 아희의 스택/큐 아키텍처

```
26개 스택 + 1개 큐 + 확장 프로토콜

[스택0] [스택1] ... [스택25] [큐]

명령 실행 → 특정 스택/큐에서 pop → 연산 → push
```

### 약속의 Python 기반 설계

```
약속 코드 → 파이썬 코드 생성 → Python 인터프리터 실행

장점: Python의 모든 라이브러리 활용 가능
      빠른 프로토타이핑
```
