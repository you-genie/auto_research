모델 제공사 SDK벤더가 직접 만든 공식 하네스 · 서로 정반대의 철학

OpenAI Agents SDK

OpenAI · Swarm(2024) 계보
모델
OpenAI + LiteLLM로 100+ 프로바이더 (락인 없음)
멀티
Handoff(라우팅) / agents-as-tools
특징
Sessions · Guardrails(tripwire) · 내장 트레이싱 · 음성/Realtime

"여러 가벼운 에이전트를 오케스트레이션한다"는 철학. Agents·Handoffs·Tools·Guardrails 4개 프리미티브로 최소한의 추상화. 단, 비주얼 도구 AgentKit(Agent Builder)은 2026-11 종료 예정.

＋최소 추상화 + 내장 트레이싱/가드레일, 프로바이더 이식성
＋음성·실시간 에이전트 지원이 업계 최강
－파일시스템/장기 자율 작업은 Claude SDK보다 약함

Claude Agent SDK

Anthropic · 구 Claude Code SDK (2025-09 개명)
모델
Claude 전용 (API/Bedrock/Vertex/Foundry 인증)
멀티
서브에이전트(격리 컨텍스트) / Dynamic Workflows
특징
내장 툴(Bash·Edit·Grep…) · Hooks · Permissions · 네이티브 MCP

"한 에이전트에게 컴퓨터를 통째로 준다"는 정반대 철학. Claude Code의 에이전트 루프·컨텍스트 관리를 그대로 코드에서 호출. 2026-06 Dynamic Workflows로 수백 개 서브에이전트 병렬 확장.

＋자율 코딩·문서·SRE 등 환경 조작형 작업 최강, 설정 제로
＋네이티브 MCP + 컨텍스트 관리 + 서브에이전트 격리
－Claude 모델 락인, 토큰 비용 높음, 음성 스토리 약함
그래프·오케스트레이션명시적 상태·그래프로 제어권을 개발자에게

LangGraph

LangChain Inc. · ~126K★
모델
StateGraph — 노드/엣지 + 조건부 분기·루프
지속성
노드마다 체크포인트 → 크래시 복구·타임트래블
멀티
supervisor / swarm 라이브러리 · Send 병렬

가장 낮은 수준의 명시적 제어. 2025-10 LangChain과 함께 1.0 GA. Klarna·Uber·LinkedIn 등 프로덕션 검증이 가장 많음. LangSmith(관측)·LangGraph Platform(배포)은 유료 레이어.

＋지속성·상태·HITL·리플레이 최강, 복잡한 제어 흐름에 최적
－장황함(CrewAI 20줄 = LangGraph 50줄+), API 잦은 변경

Google ADK

Google · Cloud Next 2025 공개
모델
Agent + Workflow(DAG) · Task API 위임
표준
네이티브 A2A(Agent Card) + MCP
배포
Vertex AI Agent Engine 원클릭 · Cloud Run

2.0(2026-03)에서 대화형 → 결정론적 그래프 워크플로우로 전환. 5개 언어(Py·Java·TS·Go·Kotlin) 지향. "에이전트 개발을 소프트웨어 개발처럼."

＋결정론적 워크플로우 + 네이티브 A2A/MCP + 멀티언어·모델
－Google Cloud/Vertex 중력, 잦은 API 변경(1.0→2.5, 14개월)

Microsoft Agent Framework

Microsoft · 2026-04 1.0 GA
통합
AutoGen + Semantic Kernel 수렴 (둘 다 유지보수 모드)
모델
Foundry·Azure·OpenAI·Claude·Bedrock·Gemini·Ollama
표준
MCP + A2A + AG-UI · YAML 선언형

.NET과 Python 동시 지원 — 에이전트 프레임워크 중 .NET 스토리가 가장 강력. sequential·concurrent·handoff·group chat·Magentic-One 패턴 내장.

＋엔터프라이즈급, Azure/Foundry 통합, 개방 표준(MCP/A2A)
－신생(두 코드베이스 병합 후 정착 중), Azure 편향
역할·대화형 멀티에이전트"AI 팀" 메타포로 협업을 표현

CrewAI

CrewAI Inc. · ~54K★ · LangChain 독립
모델
Crews(역할/목표/작업) + Flows(이벤트 기반 결정론)
메모리
단기·장기·엔티티 메모리
상용
AMP Cloud (비주얼 에디터·코파일럿·거버넌스)

역할/목표/백스토리 추상화로 몇 줄만에 크루 가동 — 램프업이 가장 빠름. 순수 자율 크루의 신뢰성 문제는 Flows로 보완. Fortune 500의 ~60%가 사용한다고 주장.

＋역할 기반 멀티에이전트 앱 최단 셋업, 대형 커뮤니티
－추상화가 블랙박스화되기 쉬움, 엔터프라이즈 실행당 과금 부담

AG2 (구 AutoGen 포크)

커뮤니티 · 2024-11 포크 · ~4.5K★
모델
AssistantAgent·UserProxy·GroupChat 대화형
상태
v1.0.0b0 (2026-07) — 1.0 향해 진행 중
거버넌스
Apache 2.0 · 오픈 거버넌스 "AgentOS"

MS의 방향이 갈리자 커뮤니티가 AutoGen을 포크. 클래식 대화형 멀티에이전트 모델을 "살아있는 프로젝트"로 계승. 오리지널 microsoft/autogen은 유지보수 모드로 사실상 종료.

＋벤더 중립, AutoGen 프로그래밍 모델을 계속 쓰고 싶다면 정답
－MS 백킹 상실, 커뮤니티 규모·리소스 제한적
데이터·RAG 중심문서·검색 파이프라인의 명가

LlamaIndex Workflows

LlamaIndex Inc. · 구 GPT Index
모델
@step 이벤트 기반 · AgentWorkflow 사전 구성
강점
LlamaHub 커넥터 · 인덱싱 · 쿼리/검색 엔진
배포
llama-deploy로 워크플로우를 서비스화

"RAG 프레임워크"에서 데이터+멀티에이전트 오케스트레이션으로 리브랜딩. Python·TS 이중 지원. 문서 중심 에이전트라면 사실상 표준.

＋RAG·문서 중심 최강, 세밀한 오케스트레이션 제어
－패키지 난립·표면적 큼, 단순 에이전트엔 과함
타입세이프·경량 (Python)신뢰성과 최소주의

Pydantic AI

Pydantic 팀 · ~16.5K★
모델
Agent + capability(툴·훅·설정 합성 단위, v2.0)
강점
Pydantic 검증·구조화 출력·의존성 주입 · 내구성 에이전트
관측
Logfire/OpenTelemetry 네이티브 · GEval

2026-06 v2.0 안정화. 타입세이프가 핵심 셀링포인트 — FastAPI/Pydantic 사용자에게 자연스러움. 검증된 구조화 출력이 필요한 프로덕션 앱에 강력.

＋신뢰성·타입세이프·구조화 출력·프로덕션 관측
－Python 전용, 2.x 빠른 릴리스로 API 변동, 멀티에이전트는 덜 성숙

smolagents

Hugging Face · ~1,000줄
모델
CodeAgent — 액션을 실행 가능한 Python 코드로 표현
실행
E2B·Modal·Docker 샌드박스
모델
transformers·Ollama·Hub·LiteLLM — 완전 자유

JSON 툴콜 대신 코드 스니펫으로 행동 — 표현력 높은 제어 흐름. 라이브러리 전체를 읽을 수 있을 만큼 투명. Hub에서 툴/에이전트 공유. 멀티모달 입력 지원.

＋단순·투명, 모델 자유, Hub 통합, 표현력 있는 코드 에이전트
－코드 실행 = 샌드박스 필수, 프로덕션 배포·내구성 기능 얇음
TS·웹 & 클라우드풀스택 개발자 경험 & 프로덕션 배포

Mastra

Mastra (Gatsby 출신) · Replit·SoftBank 사용
모델
Agents · Graph Workflows(내구성·재개) · Memory · Tools
타입
Zod 스키마로 전면 타입세이프
DX
로컬 Developer Studio 플레이그라운드

TypeScript 네이티브 종합 프레임워크. 메모리(작업+시맨틱)·내구성 워크플로우·MCP(클라이언트+서버 저작)·평가 내장. JS/TS 팀의 선두 옵션.

＋JS/TS 팀 최강 DX, 메모리+내구성 워크플로우+플레이그라운드
－TS/JS 한정, 상대적으로 신생, 엔터프라이즈 기능은 비공개

Strands Agents

AWS · 2026 Python/TS 1.0 · Amazon Q 내부 사용
모델
모델 주도형 — 모델·툴·프롬프트만 주면 루프를 모델이 주도
멀티
SubAgent · A2A · 원격 세션 매니저
모델
Bedrock·Claude·Llama·Ollama·LiteLLM

하드코딩된 그래프 대신 모델이 추론/툴 루프를 주도. MCP를 양쪽 언어에서 1급으로 지원. AWS 내부(Amazon Q·Glue)에서 검증. Bedrock AgentCore로 배포.

＋프로덕션 검증, MCP·모델 유연성 우수, Python+TS 이중
－AWS/Bedrock 중력, 커뮤니티 통합은 아직 신생

Vercel AI SDK

Vercel · ~11.5M weekly npm · ~23.7K★
모델
generateText/streamText/Object + Agent 클래스 루프
UI
useChat · AI Elements 프리빌트 컴포넌트
2026
Workflows(내구성)·Sandbox·SSE 스트리밍

＋UI/스트리밍 DX 최강, 프로바이더 중립, 대규모 채택
－v5→v6 잦은 파괴적 변경, 무거운 멀티에이전트·메모리는 약함

Q. 복잡한 상태·장기 실행·사람 승인(HITL)이 핵심인 프로덕션 에이전트

LangGraphMS Agent Framework

Q. 코드를 짜고 파일·쉘을 조작하는 자율 코딩/SRE/문서 에이전트

Claude Agent SDK

내장 툴·컨텍스트 관리·서브에이전트가 즉시. Claude 락인과 비용은 감수.

Q. 여러 모델을 자유롭게 갈아끼우며 가볍게 라우팅·음성 에이전트

OpenAI Agents SDK

최소 추상화 + 내장 트레이싱, LiteLLM로 100+ 모델, 실시간/음성 최강.

Q. 몇 줄로 "역할 나눈 AI 팀"을 빠르게 프로토타이핑

CrewAI

역할/목표/작업 추상화로 램프업 최단. 결정론 필요하면 Flows 병용.

LlamaIndex

LlamaHub 커넥터·인덱싱·쿼리 엔진의 데이터 혈통이 압도적.

Pydantic AI

Pydantic 검증 전면 + Logfire 관측. FastAPI 스택과 궁합.

Q. Next.js/React 웹앱에 스트리밍 챗·에이전트 UI 임베드

횡단 관찰

2025→2026, 지형을 바꾼 흐름

개별 프레임워크를 넘어 시장 전체에서 반복되는 패턴입니다.

01

MCP가 사실상 표준

Anthropic이 만든 MCP를 이제 거의 모든 프레임워크가 툴 연동 표준으로 채택. "어느 SDK를 쓰든 MCP 서버는 재사용 가능"이 현실이 됨.

02

A2A로 에이전트 상호운용

Google의 A2A(Agent2Agent) 프로토콜을 ADK·MS Agent Framework·Strands가 채택. 조직·런타임을 넘는 에이전트 협업이 개방 표준으로.

03

Microsoft의 대통합

AutoGen + Semantic Kernel → Microsoft Agent Framework 1.0(2026-04)으로 수렴. 두 전신은 유지보수 모드. 클래식 AutoGen은 커뮤니티 포크 AG2로 존속.

04

"결정론적 그래프"로의 회귀

순수 자율 LLM 루프의 신뢰성 한계를 겪은 뒤, LangGraph·ADK 2.0·CrewAI Flows·Mastra 모두 명시적·결정론적 워크플로우를 강화.

05

두 철학의 양극화

"많은 경량 에이전트 오케스트레이션"(OpenAI·CrewAI) vs "한 에이전트에게 컴퓨터 주기"(Claude SDK). 작업 성격이 선택을 가름.

06

오픈코어 + 유료 운영층

프레임워크 자체는 MIT/Apache로 무료, 관측·배포·거버넌스는 유료 플랫폼(LangSmith·AMP·Vertex·Foundry)으로 수익화하는 패턴 정착.
